### Come organizzare il codice
Non mettere tutta la logica nel `main`. Il `main` dovrebbe solo:
- Avviare il programma
- Creare la finestra (`JFrame`)
- Inizializzare il pannello di gioco (`JPanel` personalizzato)
- Far partire il loop del gioco

### Struttura consigliata
- **Classe Main**  
  - Crea la finestra (`JFrame`) e aggiunge il pannello di gioco.
- **Classe GamePanel (extends JPanel)**  
  - Qui disegni la grafica (`paintComponent(Graphics g)`).
  - Qui gestisci input da tastiera/mouse.
  - Qui fai girare il "game loop" (aggiornamento e ridisegno).
- **Classi di logica/dati**  
  - Esempio: `Player`, `Enemy`, `Bullet`, ecc.
  - Queste classi contengono posizione, velocit√†, stato, e metodi per aggiornarsi.
  - Non devono sapere nulla di Swing: si limitano a gestire i dati.
- **Classe GameLogic** (opzionale)  
  - Coordina gli oggetti, controlla collisioni, aggiorna lo stato del gioco.

### Esempio schematico
```java
public class Main {
    public static void main(String[] args) {
        JFrame frame = new JFrame("Il mio gioco");
        GamePanel panel = new GamePanel();
        frame.add(panel);
        frame.setSize(800, 600);
        frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
        frame.setVisible(true);
    }
}

class GamePanel extends JPanel implements Runnable {
    private Player player;

    public GamePanel() {
        player = new Player(100, 100);
        new Thread(this).start(); // game loop
    }

    @Override
    protected void paintComponent(Graphics g) {
        super.paintComponent(g);
        player.draw(g);
    }

    @Override
    public void run() {
        while (true) {
            player.update();
            repaint();
            try { Thread.sleep(16); } catch (InterruptedException e) {}
        }
    }
}

class Player {
    int x, y;
    public Player(int x, int y) { this.x = x; this.y = y; }
    public void update() { x++; } // esempio: si muove
    public void draw(Graphics g) { g.fillRect(x, y, 50, 50); }
}
```

### In sintesi
- **Swing** va bene per iniziare e imparare concetti di grafica e game loop.
- **Main** serve solo per avviare.
- **Classi separate** gestiscono dati e logica, mentre il pannello Swing si occupa di disegnare e ricevere input.
- Mantieni la grafica separata dalla logica: √® una buona abitudine che ti servir√† anche in progetti pi√π grandi.
